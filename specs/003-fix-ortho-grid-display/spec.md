# Feature Specification: 各IDのfraction表示とOrtho Viewエッジ表示修正

**Feature Branch**: `003-fix-ortho-grid-display`
**Created**: 2026-03-03
**Status**: Draft
**Input**: User description: "各IDのfractionを表示。ortho viewのときのedgeが一定間隔になっていない（グリッドが歯抜けに見える）。"

## Clarifications

### Session 2026-03-03

- Q: 「edge」はgridHelper（グリッド線）とボクセル表面のエッジハイライト（シェーダー）のどちらを指すか？ → A: ボクセル表面のエッジハイライト（シェーダー）が対象

## User Scenarios & Testing _(mandatory)_

### User Story 1 - 各ボクセルIDの占有割合（fraction）を確認する (Priority: P1)

ユーザーがボクセルモデルを開いた際、各ボクセルID（1〜15）がデータ全体に対してどの程度の割合を占めているかを視覚的に確認できる。これにより、モデル内の材料分布やボクセル構成を素早く把握できる。

**Why this priority**: ボクセルデータの統計情報はモデル理解の基本であり、ユーザーが最も頻繁に参照する情報。現状では全く表示されていないため、新規機能として最優先。

**Independent Test**: .leSファイルを開き、カラータブに各IDの個数と割合が表示されることを確認する。

**Acceptance Scenarios**:

1. **Given** ボクセルデータが読み込まれた状態, **When** カラータブを確認する, **Then** 各ボクセルID（1〜15）について、そのIDのボクセル数と全非空ボクセルに対する割合（%）が表示される
2. **Given** ボクセルデータに特定のIDが含まれていない場合, **When** カラータブを確認する, **Then** そのIDは0個、0%として表示される
3. **Given** ボクセルデータが読み込まれた状態, **When** カラータブを確認する, **Then** 空ボクセル（ID=0）の数と全ボクセルに対する割合も表示される

---

### User Story 2 - Ortho Viewでエッジハイライトが均等に表示される (Priority: P1)

ユーザーがOrthographic（正射影）カメラモードに切り替えた際、ボクセル表面のエッジハイライト（シェーダーによる描画）が等間隔で均一に表示される。現状ではOrtho Viewにおいてエッジが「歯抜け」状態（一部のエッジが表示されない、または間隔が不均一）になっており、ボクセル境界の把握が困難。

**Why this priority**: 既存機能の不具合修正であり、Ortho Viewの実用性に直結するため、P1として扱う。

**Independent Test**: .leSファイルを開き、Pキーでorthoモードに切り替え、Eキーでエッジハイライトを有効にし、エッジが等間隔で隙間なく表示されることを確認する。

**Acceptance Scenarios**:

1. **Given** ボクセルモデルが表示されている状態, **When** Ortho Viewに切り替えてエッジハイライトを有効にする, **Then** ボクセル境界のエッジが等間隔で均一に表示される
2. **Given** Ortho Viewでエッジハイライトが表示されている状態, **When** カメラをズームイン・ズームアウトする, **Then** エッジの間隔が均一なまま維持される
3. **Given** Perspective Viewでエッジハイライトが表示されている状態, **When** Ortho Viewに切り替える, **Then** エッジ表示が破綻せず均一な状態に遷移する

---

### Edge Cases

- 全ボクセルが同一IDの場合、そのIDが100%と表示される
- 空のデータ（全てID=0）の場合、非空ボクセル数が0として適切に表示される
- 非常に大きなボクセルデータ（1024³）でもfraction計算がパフォーマンスに影響しない
- Ortho Viewで極端なズームレベル（非常に近い・非常に遠い）でもエッジが均一に表示される
- カメラの回転角度にかかわらず、Ortho Viewのエッジハイライトが均一に表示される
- Perspective Viewでの距離ベースのエッジフェードは既存動作のまま維持される

## Requirements _(mandatory)_

### Functional Requirements

- **FR-001**: カラータブにおいて、各ボクセルID（0〜15）のボクセル数を表示しなければならない
- **FR-002**: カラータブにおいて、各ボクセルID（1〜15）の非空ボクセルに対する割合（%）を表示しなければならない
- **FR-003**: ID=0（空ボクセル）については、全ボクセル数に対する割合を表示しなければならない
- **FR-004**: fractionの表示は小数点以下1桁まで表示しなければならない（例: 25.3%）
- **FR-005**: Orthographic Viewにおいて、ボクセル表面のエッジハイライトが等間隔で表示されなければならない
- **FR-006**: Orthographic Viewにおいて、ズーム操作後もエッジハイライトの均一性が維持されなければならない
- **FR-007**: Perspective ViewからOrthographic Viewへの切り替え時、エッジハイライト表示が正しく遷移しなければならない
- **FR-008**: Perspective Viewにおけるエッジハイライトの既存動作（距離ベースのフェード等）は変更しない

### Key Entities

- **ボクセル統計情報**: 各ID（0〜15）のボクセル数、全体に対する割合。ボクセルデータ読み込み時に一度計算される
- **エッジハイライト**: シェーダーで描画されるボクセル境界の強調表示。カメラ投影モード（Perspective / Orthographic）に応じた適切な描画が必要

## Success Criteria _(mandatory)_

### Measurable Outcomes

- **SC-001**: ユーザーが各ボクセルIDの割合をカラータブで即座に確認できる
- **SC-002**: Ortho Viewにおけるエッジハイライトが全て等間隔で表示され、歯抜けが発生しない
- **SC-003**: Perspective / Ortho両モードでエッジハイライトが正常に機能する
- **SC-004**: 200³ボクセルデータにおいて、fraction計算がビューアーの初回表示性能（5秒以内）に影響を与えない

## Assumptions

- fraction表示は既存のカラータブ（ColorsTab）に統合する。新しいタブやパネルは追加しない
- fraction計算はボクセルデータ受信時に一度だけ行い、表示のたびに再計算しない
- エッジハイライトの修正はフラグメントシェーダー（voxel.frag）内のエッジ描画ロジックの改善で対応する
- 表示する割合の分母は、FR-002では非空ボクセル数、FR-003では全ボクセル数とする
- Perspective Viewのエッジ動作（距離ベースフェード等）は既存のまま維持する
